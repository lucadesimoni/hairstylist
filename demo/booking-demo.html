Jeder Schritt tauscht Daten über die hairlist-API aus — tippen Sie auf eine Zeile für Request/Response. In dieser Vorschau beantwortet ein eingebauter Mock die Aufrufe; auf Ihrer Website mit hinterlegtem API-Key gehen dieselben Aufrufe an die echte hairlist-API.

Konfigurieren

So funktioniert's im Live-Betrieb: Auf Ihrer Website bettet das Widget die echte hairlist-Buchungsseite ein (Modal oder inline) — inklusive Buchung, Registrierung und Anmeldung. Mit hinterlegtem Shop-API-Key kann es Termine und Konten auch nativ über die hairlist-API anlegen — genau dieser Ablauf ist hier als Demo nachgebaut.
